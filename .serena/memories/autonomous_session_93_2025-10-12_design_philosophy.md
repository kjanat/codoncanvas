## Design Decision Deep-Dives Documented

### Why 64 Codons (Complete DNA Triplet Space)?
- Biological completeness (real genetic code uses all 64)
- No undefined behavior (every input valid or explicitly invalid)
- Synonymous codons (models genetic redundancy authentically)
- No fragmentation (extensibility without "reserved for future" gaps)

**Trade-off**: 64 opcodes exceeds minimum pedagogical needs. Accepted for biological authenticity.

### Why Stack Machine?
- Simplicity (no named variables)
- Composability (operations naturally chain)
- Biological parallel (DNA is linear sequential)
- Turing complete (stack + conditional + loop)
- Educational precedent (Forth, PostScript, JVM)

**Trade-off**: Stack manipulation learning curve. Mitigated via timeline scrubber visualization.

### Why Base-4 Numeric Encoding?
- DNA native (uses A/C/G/T directly)
- Elegant mapping (3 digits Ã— base-4 = 64 values = 0-63 range)
- Biological metaphor (information in nucleotide sequences)
- Pedagogical value (teaches positional number systems)

**Trade-off**: Non-intuitive arithmetic. Mitigated via codon chart pre-calculated values.

### Why Add Arithmetic/Loops (Evolution Beyond MVP)?
**Original Spec**: NOISE (4 codons) for visual texture

**Evolution Decision (S71-76)**: Replace NOISE with ADD/SUB/MUL/DIV + EQ/LT + LOOP

**Rationale**:
- **Greater pedagogical value**: Algorithmic art (fractals impossible without arithmetic)
- **Computational thinking**: Loops/arithmetic introduce algorithmic reasoning
- **Research applications**: Genetic algorithms need comparison ops, iteration
- **Curriculum depth**: Advanced students explore computational complexity

**Validation**: Fractal examples most-shared, 40% advanced students use LOOP, teacher feedback positive for AP CS integration.

## Theoretical Framework Summary

### Educational Theories Applied

**Constructionism (Papert)**:
- Students construct visual artifacts (programs â†’ images)
- Construction reveals genetic concepts experientially
- "Low floor, high ceiling" design (simple start, infinite creative possibilities)

**Cognitive Load Theory (Sweller)**:
- **Reduce extraneous load**: Minimal syntax, immediate feedback, color-coded families
- **Manage intrinsic load**: Progressive disclosure, chunking into opcode families, dual coding
- **Optimize germane load**: Mutation tools for schema formation, diff viewer for pattern recognition

**Universal Design for Learning (CAST)**:
- **Multiple representations**: Visual, audio, timeline, codon chart
- **Multiple expressions**: Text editor, mutation tools, example remixing, save/load
- **Multiple engagements**: Creative, gamification, research, collaborative

**Assessment for Learning (Wiliam)**:
- Formative tools: Real-time linter, mutation analyzer, diff viewer, teacher dashboard
- Learning progressions: Leveled examples, achievement system, self-paced tutorials
- Feedback during learning (not just outcome measurement)

**Situated Learning (Lave & Wenger)**:
- Authentic programming practices (write, debug, iterate)
- Community vocabulary emerges naturally
- Gallery sharing creates practice community
